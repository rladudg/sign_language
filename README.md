# 청각장애인용 화상통화 보조 시스템

수화와 음성을 실시간으로 텍스트로 변환해 한 화면에 함께 표시하는 화상통화 보조 시스템입니다. 수화로 검출된 개별 자모는 한글 오토마타로 완성형 글자로 조합하고, 음성은 Google STT로 변환해 청각장애인과 비장애인 간 양방향 소통을 지원합니다. 화상통화 영상은 기존 솔루션 연동을 가정하며, 본 시스템은 그 위에 얹히는 **실시간 자막 통합**을 담당합니다.

## 사용 기술

| 구분 | 기술 |
|------|------|
| UI / 통신 | C# WinForms, TCP/IP Socket |
| 음성 처리 | Python, SpeechRecognition (Google STT) |

## 주요 기능

**멀티스레드 TCP 서버** 수화·음성 패킷을 별도 스레드에서 실시간 수신하고, `Invoke`로 UI 스레드에 안전하게 반영해 수신 도중에도 화면이 멈추지 않도록 했습니다.

**한글 오토마타 직접 구현** 수화로 검출된 개별 자모(초·중·종성)를 완성형 한글로 조합하는 로직을 라이브러리 없이 구현했습니다. 삭제·초기화 명령도 오토마타 상태에 맞춰 처리합니다.

**음성 STT 모듈 (Python)** 마이크 입력을 Google STT로 텍스트 변환한 뒤, 글자 단위로 `VOICE:` 패킷을 만들어 TCP로 전송합니다. 음성인식은 별도 스레드(daemon)로 돌려 카메라 미리보기와 분리했고, `adjust_for_ambient_noise`로 주변 소음을 보정합니다.

**두 입력의 단일 화면 통합** 수화·음성 입력을 하나의 화면에 합치고, 처리 상태(Ready / Detecting / Hold / Confirmed / Error)와 실시간 로그 UI로 동작을 한눈에 확인할 수 있게 구성했습니다.

## 통신 프로토콜

텍스트 기반의 단순한 패킷 프로토콜로 설계했습니다.

| 패킷 | 의미 |
|------|------|
| `CHAR:{자모}` | 수화로 검출된 개별 자모 (오토마타로 조합) |
| `VOICE:{글자}` | 음성 STT 변환 결과 |
| `CMD:DELETE` | 한 글자 삭제 |
| `CMD:CLEAR` | 전체 초기화 |
| `STATE:{상태}` | 처리 상태 표시 |

## 시스템 구조

```
[수화 인식 (팀원, Python)] ──┐
                            ├─→ TCP 패킷 ─→ [C# WinForms 서버] ─→ 자모 조합 / 자막 표시
[음성 STT (Python)]      ──┘
```

수화 인식 결과는 `CHAR:` 패킷으로, 음성 변환 결과는 `VOICE:` 패킷으로 들어오며, C# 서버가 두 입력을 한 화면에서 수신·처리합니다.

## 담당 역할

통신 · 조합 · UI 파트를 담당했습니다.

- C# WinForms 멀티스레드 TCP 서버 구축 (패킷 수신 및 UI 스레드 안전 처리)
- 개별 자모를 완성형 한글로 조합하는 한글 오토마타 구현
- Python 기반 음성 STT 모듈 개발 (Google STT, 스레드 분리, TCP 전송)
- 두 입력을 단일 화면에 통합, 상태 표시 및 로그 UI 구성

> 수화 인식 모델은 팀원이 담당했으며, 본 파트는 전달된 자모 패킷의 수신·조합·표시를 담당했습니다.
